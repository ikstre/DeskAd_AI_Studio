## 작업 내용

- 로그인/회원가입 UI 방향을 검토하고, 기존 중앙 카드형 화면을 브랜드 패널 + 인증 카드 구조로 변경했다.
- 로그인 화면에서 일반 앱 사이드바가 보이지 않도록 로그인 전용 스타일을 추가했다.
- 왼쪽 영역에는 DeskAd AI Studio 소개, 3D 셋업/광고 문구/포스터 제작 기능 요약 카드를 배치했다.
- 오른쪽 영역에는 로그인/회원가입 탭과 안내 문구를 유지해 기존 인증 흐름을 그대로 사용할 수 있게 했다.
- Streamlit 컴포넌트가 HTML 래퍼 사이에 끼면 레이아웃이 깨질 수 있어, 최종 구조는 `st.columns`와 `st.container(border=True)` 기반으로 정리했다.
- 새로고침 시 로그인 상태가 초기화되는 문제를 수정했다.
- `origin/main` 최신 변경 사항을 현재 브랜치에 병합하고, HyperCLOVA grid_three 타임아웃/작업 큐/진행률 관련 충돌을 정리했다.
- URL query param에 인증 토큰이 노출되는 문제를 확인하고, Streamlit 로그인 화면에서는 auth URL 이동을 사용하지 않도록 정리했다.
- 로그인 성공 후에는 Streamlit `session_state`에 세션 토큰을 저장하고 즉시 메인 화면으로 전환하도록 했다.
- FastAPI에는 `/auth/session`, `/auth/cookie-code`, `/auth/cookie`, `/auth/clear-cookie` 엔드포인트가 남아 있지만 현재 Streamlit 로그인 UI에서는 `/auth/cookie` 리다이렉트를 호출하지 않는다.
- Streamlit은 기존 HttpOnly cookie가 있는 경우에만 `st.context.cookies`와 `/auth/session`으로 세션 복원을 시도한다.
- PR 제출 전 UI 완성도를 높이기 위해 Step 4 광고 콘텐츠 화면을 추가 조정했다.
- 광고 콘텐츠 단계에서 결과 미리보기 영역이 너무 좁아 보이던 문제를 줄이기 위해 미리보기/편집 패널 비율을 조정했다.
- Grid 3컷 광고 카드 미리보기가 단순 텍스트처럼 보이지 않도록 제품/공간/포인트 3개 컷 카드 구조로 개선했다.
- 사이드바의 고급 설정/진단 영역에서 호버 전 글자가 흐리게 보이는 문제를 줄이기 위해 expander 내부 텍스트 색상 대비를 고정했다.
- Streamlit custom component iframe에서 자동 top-level 이동이 브라우저 보안 정책에 막히는 문제를 확인했다.
- 로그인 성공 후 추가 버튼 없이 Streamlit 세션 상태를 즉시 갱신하고 메인 화면으로 넘어가도록 수정했다.
- Streamlit component iframe에서 top-level 자동 이동은 사용자 클릭으로 인정되지 않아 HttpOnly cookie 저장 자동화는 별도 구조 변경이 필요함을 확인했다.
- Streamlit 프론트에서 더 이상 사용하지 않는 `/auth/cookie-code`, `/auth/cookie`, `/auth/clear-cookie` URL 생성 헬퍼를 `ui/api_client.py`에서 제거했다.
- 로그인 첫 화면을 넷플릭스식 좌측 대형 hero copy + 우측 로그인/회원가입 카드 구조로 재조정했다.
- 왼쪽 hero 영역은 서비스 설명을 짧고 강한 문구로 바꾸고, 3D 셋업/문구 후보/포스터 검수 핵심 기능을 pill과 카드로 요약했다.
- 로그인/회원가입 아이디 정책을 영문과 숫자만 허용하도록 프론트 검증, 백엔드 회원가입 검증, 테스트를 함께 수정했다.
- 비밀번호 입력 중 CapsLock이 켜진 경우 안내 문구를 보여주는 브라우저 이벤트 기반 보조 컴포넌트를 추가했다.
