## 추가 작업

- `/auth/signup` 응답 모델을 로그인 응답 모델과 분리하기 위해 `SignupResponse`를 추가했다.
- 회원가입은 성공 시 자동 로그인 토큰을 발급하는 기존 흐름을 유지하므로, `SignupResponse`는 `LoginResponse`를 상속해 같은 세션 응답 필드를 사용하도록 정리했다.
- `backend/main.py`의 `/auth/signup` 엔드포인트 `response_model`을 `LoginResponse`에서 `SignupResponse`로 변경했다.
- 로그인/회원가입 기능 구현 상태를 확인했다. 백엔드 인증 엔드포인트, PBKDF2 사용자 저장소, 가입 코드 검증, Streamlit 로그인 게이트, UI API 호출부가 이미 연결되어 있어 신규 기능 추가는 필요하지 않았다.
- `SignupResponse` 내부의 설명용 error 코드 주석은 기능에 필요 없는 중복 설명이라 제거했다.
